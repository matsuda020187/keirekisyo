# 認証

## AUTH001 ログイン

SSO（Azure AD／Google／GitHub）でログインする。ID/パスワードによるログインは提供しない。

| 項目 | 種別 | 必須 | 内容 | 備考 |
|---|---|---|---|---|
| Microsoftアカウントでログイン | ボタン | - | Azure AD (OIDC) へリダイレクト | |
| Googleアカウントでログイン | ボタン | - | Google OAuthへリダイレクト | |
| GitHubアカウントでログイン | ボタン | - | GitHub OAuthへリダイレクト | 個人アカウントで会社メールと紐付いていない場合は未登録エラーになる |

**エラー表示（条件付き）**
- 未登録: 認証プロバイダから取得した確認済みメールアドレスが`user_account.email`に該当レコードなし →「このメールアドレスは登録されていません。管理者に新規登録を依頼してください。」
- 退職済み: 該当employeeの`employment_status = 2（退職）` →「このアカウントは無効化されています。心当たりがない場合は管理者にお問い合わせください。」
- プロバイダ不一致: 登録済みの`auth_provider`と異なるプロバイダで認証された（メールは一致するが`external_id`が不一致の場合を含む）→「このアカウントは別のログイン方法で登録されています。初回に使用したログイン方法をお使いください。」ログインは実行しない

**初回ログイン後の遷移（ロール別）**: 一般社員／管理職は`is_registered=false`ならEDT001（初回登録）へ。**人事・営業は経歴書を作成しないため、EDT001を経ずREF001（トップ）へ直行し、ログイン成立時に`is_registered`を自動でTRUE、`employee.name`にSSOの表示名を設定する**（REF007の状態表示が「初回未登録」のまま残らないようにするため）。

**プロバイダの扱い**: 1アカウントにつき、初回ログインで使用したプロバイダ（`auth_provider`）に固定される。2回目以降に別プロバイダで認証された場合は上記「プロバイダ不一致」エラーとする（切替機能は提供しない）。

## REF002 経歴書一覧

人事・営業／管理職が全社員の経歴書を検索・一覧表示する。
一般社員は自身と**同一部署**の社員（自分を含む）のみを検索・一覧表示できる（検索対象がサーバー側で自動的に限定される。所属組織フィルタも自部署とその配下のみ選択可能）。
基本条件・スキル条件・取得資格条件はそれぞれ折りたたみ可能なセクション。

**同一部署の判定ルール（REF003・REF008の閲覧権限も本ルールに従う）**: 双方の所属組織単位（`employee.organization_unit_id`）から親を遡って最初に到達する部署（`unit_level=2`。自身が部署所属ならその部署、Gr所属なら親の部署）が一致する場合に「同一部署」とする。事業部直下所属・未所属（NULL）の社員は部署を持たないため、一般社員の同一部署閲覧では見る側・見られる側のどちらにもならない（本人の経歴書のみ閲覧可／人事・営業・管理職からは従来どおり閲覧可）。

| 項目 | 種別 | 必須 | 内容 | 備考 |
|---|---|---|---|---|
| 氏名・カナ | テキスト | 任意 | 50文字以内 | 部分一致検索 |
| 所属組織 | 階層選択 | 任意 | 事業部／部署／Grから選択 | 上位を選ぶと配下も含めて検索。一覧表示は最下層のみ |
| 経験年数（下限〜上限） | 数値 | 任意 | 0〜99の整数 | 下限≦上限 |
| スキル条件 | 複数選択＋サジェスト | 任意 | スキルマスタから複数選択 | AND/OR切替あり（トグル） |
| 取得資格条件 | 複数選択＋サジェスト | 任意 | 資格マスタから複数選択 | AND/OR切替あり（トグル） |
| 検索結果一覧 | 表示 | - | 氏名／所属組織／経験年数／主なスキル | 「詳細」でREF003へ。退職者はデフォルトで一覧から除外し、フィルタで表示切替可能とする |

**AND/OR切替の挙動**: 画面上に説明文は常時表示しない。トグルにマウスオーバーした際にツールチップで「AND：選択した全スキルを持つ人を検索／OR：いずれか1つでも持つ人を検索」を表示する。

# 共通部品

## CMN001 削除確認モーダル

一覧・編集画面から削除操作を行う際に表示する確認ダイアログ。

| 項目 | 種別 | 内容 |
|---|---|---|
| 確認メッセージ | 表示 | 呼び出し元から指定可能。デフォルトは「削除してもよろしいですか？」＋対象名（EDT007の退職処理・現職復帰は専用文言を指定） |
| キャンセルボタン | ボタン | モーダルを閉じる |
| 実行ボタン | ボタン（危険操作） | デフォルトは論理削除（deleted_atをセット）。呼び出し元により動作が異なる（EDT007参照） |

**マスタ削除時の参照チェック**: MST001〜005からの削除では、実行前に対象マスタへの参照有無をチェックし、参照が存在する場合は削除を実行せずエラー「使用中のため削除できません」を表示する（schema.md 一般ルール参照）。

呼び出し元: EDT005（プロジェクト経歴の削除）、MST001〜005（マスタ一覧の削除アクション全般）、EDT007（退職処理）。EDT007の退職処理は本モーダルを使うが、動作は論理削除ではなく`employment_status`の更新である点に注意。

※EDT003（スキル）・EDT004（資格）の明細行の削除は、保存前の画面内操作のため本モーダルは使わない（保存時に確定）。
