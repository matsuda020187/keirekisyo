# REF008 スキルマップ／組織ダッシュボード v3 拡張案

`screens.md` の REF008 仕様（組織単位選択・スキル別/資格別保有者数集計・保有者名展開）を正式仕様として維持したまま、
その上に追加実装した拡張機能をまとめたもの。参照元のイメージ（`skill_dashboard_v3_mockup.html`）を元にした
プロトタイプで、**`screens.md`・`decisions.md`など既存の設計資料は変更していない**。

正式仕様への昇格要否・追加機能の要不要は別途判断すること。

## 追加した機能

既存実装（組織単位選択・スキル別/資格別保有者数・保有者名展開）に加えて以下を追加。
閲覧権限（保有者名の表示制御）は既存のREF002同一部署判定ルールをそのまま踏襲し、新機能にも一貫して適用する。

| 機能 | 内容 |
|---|---|
| サマリー統計カード | 登録メンバー数／登録スキル種類数／資格保有件数／保有者1名のみのスキル数を表示 |
| 検索 | スキル名・資格名・保有者名（閲覧可能な範囲のみ）で絞り込み。既存の組織選択と同じGETフォームに統合（`q`パラメータ） |
| 資格カテゴリタブ＋ドーナツ | 資格カテゴリ（マスタ管理の資格カテゴリ）ごとにタブ切替し、内訳をドーナツ＋凡例で表示 |
| スキルカテゴリタブ＋上位10件 | スキルカテゴリごとにタブ切替し、保有者数の多い順に表示。11件目以降は「すべて表示」で展開 |
| 部署×スキルカテゴリ ヒートマップ | 選択中の組織スコープ内の部署（unit_level=部署）を行、スキルカテゴリを列とし、保有者の実人数（重複なし）を色の濃淡で表示 |
| 属人化リスク一覧 | 保有者が1名のみのスキルを一覧表示（スキル名・保有者・所属部署）。氏名表示は閲覧権限に連動 |
| Excel出力 | 資格別保有者数・スキル別保有者数（全件）・属人化リスク一覧をそれぞれExcelファイルとしてダウンロード |

## 実装上の決定事項

- **データモデルの変更なし**。既存テーブル（`employee`/`employee_skill`/`skill`/`skill_category`/
  `employee_certification`/`certification`/`certification_category`/`organization_unit`）の集計のみで実現しており、
  マイグレーションは発生していない。
- 集計ロジックは `lib/skill-map-data.ts` に切り出し、画面表示（`app/skill-map/page.tsx`）とExcel出力
  （`app/api/skill-map/export/[type]/route.ts`）の両方から同一関数を呼び出す（表示とExcelの数値が食い違わないようにするため）。
- ヒートマップの行（部署）は、選択中の組織スコープに実在する社員が所属する部署のみを表示する。
  事業部直下所属・未所属の社員（部署を持たない）はヒートマップの集計対象から除外する（REF002の部署未確定ルールと同じ扱い）。
- 資格カテゴリ・スキルカテゴリのタブは、マスタ管理（MST001/MST002）に登録された実際のカテゴリ名を使用する。
  モックアップの「国家資格／ベンダー資格／その他」は固定タブではなく、実データのカテゴリがそのままタブ名になる。
- Excel出力は既存の`Excel出力(準備中)`（マイページ・経歴書詳細）とは独立した新規実装。`exceljs`を新規依存として追加した
  （プロジェクト内に既存のExcel生成ライブラリがなかったため）。既存の「準備中」ボタンへの反映は本対応の範囲外。
- 検索・ヒートマップ・属人化リスクの計算コストはO(社員数×保有スキル数)。現状の想定社員数規模では問題にならないが、
  全社員数が大きく増えた場合はDB側での集計（GROUP BY）への切り替えを検討する。
