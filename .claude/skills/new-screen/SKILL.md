# 画面実装手順

<!--

## 手順

1. **仕様を読む**: `docs/screens.md` の該当画面の節を読む。
   遷移元・遷移先が関わるなら `docs/screen-flow.md` も確認する
2. **データ要件の確認**: 表示・更新するデータのテーブル定義を `docs/schema.md` で確認する
3. **実装方針の決定**(このプロジェクトの型):
   - ページは Server Component で作り、データ取得もそこで行う
   - フォームや操作は Server Actions を第一候補にする
   - インタラクティブな部分だけを `"use client"` の子コンポーネントに切り出す
4. **認可の実装**: 画面ごとに「誰がアクセスできるか」を仕様で確認し、
   ページ冒頭で `auth()` によるチェックを入れる。認可チェックのない画面を作らない
5. **テスト追加**: ロジック(Server Actions、ユーティリティ)に `*.test.ts` を追加する
6. `npm run verify` で全検証を通す

## チェックリスト(完了報告の前に自己確認)

- [ ] 仕様(docs/screens.md)との差異がない。あるなら差異を明記して人間に確認した
- [ ] 未認証・権限なしユーザーでのアクセス時の挙動が定義されている
- [ ] ローディング状態(loading.tsx)とエラー状態(error.tsx)を考慮した
- [ ] Prisma Client をクライアントコンポーネントから import していない
- [ ] verify が通っている
