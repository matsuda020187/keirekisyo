以下の画面を Playwright で実機確認してください: $ARGUMENTS

前提: 開発サーバー(npm run dev)が起動していること。起動していなければ
起動してから確認する(バックグラウンド実行)。

確認手順:

1. docs/screens.md の該当画面の節を読み、「何が表示・操作できるべきか」を把握する
2. Playwright で対象画面を開く
3. 以下を確認する:
   - 仕様どおりの要素が表示されているか
   - 主要な操作(フォーム入力、ボタン、遷移)が機能するか
   - ブラウザのコンソールにエラーが出ていないか
   - 認証が必要な画面の場合、未認証時に適切にリダイレクト/ブロックされるか
4. スクリーンショットを撮る
5. 結果を「仕様との一致点 / 差異・不具合 / コンソールエラー」の形式で報告する

注意: 確認だけを行い、この中では修正しない。不具合があれば報告して指示を待つ。
